概述

Spring是一个轻量级、非入侵式的控制反转(IoC)和面向切面(AOP)的框架。

企业级开发的标配基本就是Spring5+SpringBoot2+JDK8

Spring的优点

1.IOC和DI支持

Spring核心就是一个大的工厂容器，可以维护所有对象的创建和依赖关系，Spring工厂用于生成Bean，并且管理Bean的生命周期，实现高内聚低耦合的设计理念。

2.AOP编程的支持

Spring提供了面向切面编程，就而已方便的实现对程序进行权限拦截、运行监控等切面功能。

3.声明式事务的支持

支持通过配置来完成对事务的管理，而不需要通过硬编码的方式。以前重复的一些事务提交，回滚的JDBC代码，不需要自己写了。

4.快捷测试的支持

Spring对Junit提供支持，可以快速的通过注解测试Spring程序。

5.快速继承功能

方便集成各种优秀框架，Spring不排斥其他开源框架，其内部提供了对各种优秀框架的直接支持。

6.复杂的API模版封装

Spring对JavaEE开发中非常难用的一些API都提供了模板化的封装，这些封装使得Api的使用难度大大降低。

Spring的七大模块

  • Spring Core：Spring核心，是框架最基础的部分，提供IOC和依赖注入DI特性。
  • Spring Context：Spring上下文容器，SBeanFactory功能加强的一个子接口
  • Spring Web：提供Web应用开发的支持。
  • Spring MVC：针对Web应用中MVC思想的实现
  • Spring DAO:针对JDBC抽象层，简化了JDBC编码，编码更具有健壮性。
  • Spring ORM：支持用于流行的ORM框架的整合。
  • Spring AOP：面向切面编程，提供了与AOP联盟兼容的编程实现

Spring常见注解

WEB

  • @Controller：组合注解(组合了@Component注解)，应用在MVC层(控制层)

  • @RestController

    该注解为一个组合注解，相当于@Controller和@ResponseBody的组合，注解在类上，意味着该Controller的所有方法都默认加上了@ResponseBody。

    在Controller基础上，将Java转为RESTfulAPI控制器，处理HTTP请求和响应。
    当SpringBoot接受到HTTP请求的时候，会查找请求URL匹配的RestController控制器，将调用匹配方法并将其返回值转换为JSON或者XML响应体，并发送回客户端。

  • @RequestMapping

    用于映射Web请求的注解，包括访问路径和参数。可以用在类或者方法上面。
    用于方法上，表示在类的父路径下追加方法上注解中的地址将会访问到该方法。
    用在类上，表示类中的所有响应请求的方法都是以该地址作为父路径。
    当请求了value中的地址，Spring框架就会寻找使用了这个注解的方法或者类来处理这个请求。
    value的属性必须设置值，通过当前请求的请求地址来匹配请求。value属性是一个字符串类型的数组，可以通过设置数组，将多个请求映射到一个方法上，只需要给value来指定一个包含多个路径的数组。

    可以根据请求类型使用不同的注解：

    • @PostMapping
    • @GetMapping
    • @PutMapping
    • @DeleteMapping

  • @ResponseBody 支持将返回值放在response内，而不是一个页面，通常用户返回json数据

  • @RequestBody 允许request参数在request体中，而不是直接连接在地址后面。

  • @PathVariable 用于接受路径参数 比如@RequestMapping(”/get”)，将注解放在参数前可以获取该值，通常作为Restful的接口实现方法。

容器

  • @Component： 表示一个带注解的类是一个组件，是Spring管理的Bean。当使用基于注解的配置和类路径扫描时，这些类被视为自动检测的候选对象。是一个元注解。

  • @Service：组合注解(@Component)应用在service层

  • @Repository 组合注解(@Component) 应用在Dao层(数据访问层)

  • @Autowired Spring提供的工具(由Spring的依赖注入工具(BeanPostProcessor\BeanFactoryPostProcessor)自动注入)。

    @Resource

    是 Java 的一个注解，用于依赖注入（Dependency Injection），特别是当使用 Java EE 或 Spring 框架时。这个注解告诉容器（例如，一个 Java EE 服务器或 Spring 框架）去查找并注入一个匹配的资源。

    @Autowired和@Resource都是用于依赖注入的注解，但它们之间存在一些重要的区别。

    1. required属性：
      • @Autowired注解有一个required属性，可以设置为true或false。如果设置为true（默认值），当容器中没有找到匹配的bean时，会抛出异常。如果设置为false，容器不会抛出异常，而是将字段设置为null。
      • @Resource注解没有提供类似的required属性。如果容器中没有找到匹配的bean，它默认会抛出异常。
    2. 处理机制：
      • @Autowired和@Inject基本上是一样的，因为它们都使用AutowiredAnnotationBeanPostProcessor来处理依赖注入。
      • 然而，@Resource注解使用的是CommonAnnotationBeanPostProcessor来处理依赖注入，这是与前两者的主要区别。
    3. 默认行为：
      • @Autowired默认按类型自动注入，也就是说，如果容器中存在多个相同类型的bean，它会使用属性的名称作为bean的id去容器中查找。
      • @Resource默认按名称自动注入。如果没有指定名称，那么它将使用属性的名称作为默认名称。
    4. 使用范围：
      • @Autowired是Spring框架的一部分，主要在Spring环境下使用。
      • @Resource是Java EE的一部分，主要在Java EE环境下使用。

    @AllArgsConstructor是 Lombok 库中的一个注解，用于自动生成一个包含所有字段的构造器。当你在一个类上使用@AllArgsConstructor` 注解时，Lombok 会在编译时自动生成一个包含该类所有字段的构造器。
    可以用这个在类上，使用自动构造器构造进行构造器形式的注入。

  • @Qualifier 该注解通常跟@Autowired一起使用，当相对注入的过程做更多的控制，该注解可以帮助配置，比如两个以上类型相同的bean时Spring无法抉择

  • @Configuration 声明当前类是一个配置类(相当于一个Spring配置的xml文件)

  • @Value 可用在字段、构造器参数跟方法参数上，指定一个默认值，支持#{}和${}两个方式。一般将application.properties配置的属性值赋值给变量

  • @Bean 注解在方法上，声明在当前的方法的返回值为一个Bean，返回的Bean对应的类中可以定义init()方法和destroy()方法，然后再@Bean(initMethod=”init”, destroyMethod = “destroy“)定义，在构造之后执行init，在销毁前执行destroy。

  • @Scope 定义采用什么模式去创造Bean，设置的类型包括：Singleton、Prototype、Request、Session、GlobalSession

AOP

  • @Aspect 声明一个切面 使用注解定义建言(advice),可以直接将拦截规则(切点)作为参数。
    • @After：在方法执行之后执行
    • @Befor：在方法执行之前执行
    • @Around 在方法执行之前和之后执行
    • @PointCut 声明切点在Java配置类中使用@EnableAspectJAutoProxy 注解开启Spring对AspectJ代理的支持(类上)。

事务

  • @Transactional 在要开启事务的方法上使用@Transactional注解，即可声明式开启事务

Spring中的设计模式

  • 工厂模式 Spring容器本质上是一个大工厂，使用工厂模式通过BeanFactory、ApplicationContext创建bean对象
  • 代理模式 SpringAOP功能就是通过代理模式来实现的，分为静态代理和动态代理
  • 单例模式 Bean默认都是单例的，有利于容器对Bean的管理。
  • 模版模式 Spring中JdbcTemplate等以Template结尾的对数据库、网络等等进行操作的模板类，就使用到了模版模式
  • 观察者模式 Spring事件驱动模型就是观察者模式很经典的应用
  • 适配器模式 SpringAOP的增强或者通知使用到了适配器模式，MVC中也是用到了适配器模式适配Controller
  • 策略模式 Spring中有个Resource接口，它的不同实现类，会根据不同的策略去访问资源。
